Reporting Bugs

If something is not working as expected, we want to hear about it.

From the App

Go to Settings → Feedback & Support → Report a Bug. Your email app opens with diagnostic details already filled in. Add the information below before sending.

What to include

  • What happened — Describe the problem in plain language
  • Steps to reproduce — What you tapped or did before the issue appeared
  • Expected behavior — What you thought should happen instead
  • App version — Pre-filled when you use in-app reporting (for example, 0.6.0+2)
  • Device and OS — Pre-filled (for example, Android version or iOS version)
  • Screenshots or screen recordings — Helpful when the issue is visual
  • Backup or OCR context — If the bug relates to backup, restore, OCR, or attachments, mention which feature and any error message shown

Please do not include renewal records, policy numbers, passport details, or other sensitive vault content unless you choose to for troubleshooting—and only if you are comfortable doing so.

You can also email jayanthrajiv@gmail.com directly or use the Contact page.

Optional: crash reporting

If you have enabled Share Anonymous Crash Reports, some stability issues may already include anonymous diagnostic data. You can still email us with steps to reproduce so we can investigate faster.

Backup Issues

Renew Vault offers optional encrypted backup and restore. Your data is your responsibility to back up—loss of a device or cleared App data cannot be recovered without a backup you created.

Local export (.rvbackup)

To create a backup:

  1. Open Settings → Backup & Restore (or the Backup screen from Settings)
  2. Choose Backup Data to create an encrypted .rvbackup file
  3. Save or share the file through your preferred app

Tips:

  • New exports use encrypted .rvbackup format (AES-256-CBC). Legacy plain .json files may still be imported for migration, but new exports use .rvbackup only.
  • Store backup files somewhere safe—a cloud drive, email, or external storage you control.
  • Enable backup reminders in Settings if you want a nudge to back up on a schedule (reminders do not upload data automatically).

Google Drive backup

To back up to Google Drive:

  1. Open the Backup screen
  2. Sign in with Google when prompted
  3. Tap Backup to Google Drive

Encrypted backups are stored in a dedicated "Renew Vault Backups" folder in your Google account. Google Sign-In is used only for this feature—not for App login or vault sync.

Common issues:

Issue What to try
Sign-in fails or is cancelled Check your internet connection and Google account status; try signing out and back in from the Backup screen
No backups listed when restoring Upload at least one backup to Google Drive first from the same Google account
Upload or download fails Confirm network connectivity; ensure sufficient Google Drive storage
Wrong Google account Sign out from the Backup screen and sign in with the account that holds your backups

Restore

To restore locally:

  1. Go to Settings → Backup & Restore → Restore Data
  2. Select a .rvbackup or legacy .json file
  3. Review the summary (item count, family members, attachments)
  4. Confirm restore

To restore from Google Drive:

  1. Open Restore from Google Drive on the Backup screen
  2. Sign in with the Google account that contains your backups
  3. Select a backup and follow the on-screen steps

Common restore errors:

  • "Unsupported file type" — Select a .rvbackup or .json backup file, not another file type
  • "Unsupported backup version" — The backup may be from a newer App version; update Renew Vault and try again, or use a backup created with your current version
  • "Unable to read backup contents. The file may be corrupted." — The file may be incomplete or damaged; try another backup copy
  • "Backup appears corrupted" — Verification failed; use a different backup file if available

Restore replaces your current vault data with the backup contents. Confirm the summary before proceeding.

OCR Troubleshooting

OCR helps you scan documents and pre-fill renewal fields from photos or images. Processing runs on your device using Google ML Kit Text Recognition.

Permissions

OCR requires access to:

  • Camera — To capture a new photo of a document
  • Photos / media — To pick an existing image from your gallery

If you deny a permission, the related option will not work until you grant it in your device settings. Renew Vault shows a brief explanation before requesting permission.

Accuracy and review

OCR works best with:

  • Good lighting and a flat, in-focus document
  • Minimal glare, shadows, or cropping of important text
  • Common document types such as passports, driving licences, insurance policies, and warranties

Extracted fields may be incomplete or incorrect. Always use the review screen to edit title, category, dates, document number, and other fields before confirming. You can Retake the scan if the image quality was poor.

Raw OCR text is not stored. Only the structured fields you save—and optional correction hints to improve future scans—are kept locally.

Common OCR problems:

Issue What to try
Camera or gallery will not open Grant Camera or Photos permission in device settings
No text detected Retake with better lighting; ensure the full document is visible
Wrong dates or document type Edit fields on the review screen before saving
Scan is slow Large or high-resolution images may take longer; wait for the progress indicator to finish

Notifications Troubleshooting

Renew Vault schedules local notifications on your device based on the renewal dates and reminder settings you configure. Reminders are not sent from our servers.

Enable notifications

Make sure notifications are enabled for Renew Vault in both places:

  • In the App — Settings → Notifications
  • On your device — System settings for Renew Vault notifications

Renewal vs. backup reminders

Renewal reminders alert you about upcoming expirations based on dates and reminder settings on each item. Backup reminders, if enabled in Settings, only notify you to back up on a schedule—they do not upload data automatically.

Common notification issues:

Issue What to try
Not receiving renewal reminders Enable notifications in Settings → Notifications and in your device system settings; confirm the item has a renewal date and reminder configured
Backup reminder but no upload happened Backup reminders only notify you to back up—they do not upload data automatically; open Settings → Backup & Restore to create a backup manually

Attachment Issues

You can attach photos, images, and PDF files to renewal items for reference. Attachments are encrypted at rest on your device (AES-256-CBC).

Adding attachments

  • Camera — Capture a new photo
  • Gallery — Select an existing image
  • File picker — Choose a PDF or image file you explicitly select

The App reads only files you choose through the system picker.

Limits

Renew Vault currently allows one attachment per item on the free tier. If you reach the limit, remove an existing attachment or contact us with feedback if you need more.

Viewing attachments

When you open an attachment, the App decrypts it temporarily for preview. If a file appears missing or corrupt, the original may have been deleted from device storage outside the App, or the attachment record may not match stored files—try restoring from a backup if you have one.

Common attachment problems:

Issue What to try
Camera or gallery unavailable Grant the required permission in device settings
Cannot add another attachment Free tier allows one attachment per item; remove the existing one first
PDF will not open Confirm the file is a valid PDF and was fully saved; try re-attaching
Attachment missing after restore Ensure the backup included attachments and restore completed successfully
